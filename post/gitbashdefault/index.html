设置Windows Terminal默认终端为Git Bash

2020-08-17 742 words 2 mins read times read

Contents

    • 准备工作
    • 生成GUID

以前都是用CMD多点,最近Git Bash用的越来越多了就将Windows Terminal的默认终端换成Git Bash了,如果还没用过这两个软件,这里推荐一下,在Windows下算是很好的的组合了(仰望用Linux和Mac的同学(..•˘_˘•..)

准备工作

Windows Terminal

可以直接在Microsoft Store中下载安装

Git Bash for Windows

下载安装之后需要添加环境变量(具体方法自行Google),完成后在CMD输入[git –version] ,得到的结果应该为git version 2.*,确认无误后再进行之后的操作

1
2

git --version
git version 2.27.0.windows.1

生成GUID

什么是GUID?

GUID是"Globally Unique Identifier"的首字母缩写,它是一个128位整,用于标识资源,GUID通常用于微软的产品,而其他地方通常会称做UUID(Universally unique identifier).

打开Windows PowerSell输入以下命令我Git Bash生成一个新的guid

1

[guid]::NewGuid()

复制新生成的guid

1
2
3

Guid
----
76b973f5-c6b2-4ed2-aab5-64c2401ddf65h

将Git Bash添加进Windows Terminal

打开Windows Terminal按下快捷键[Ctrl+,]在弹出的setting.json文件中找到list列表， 把以下配置添加上去，并把guid行花括号中的guid换成自己的

{
"guid": "{76b973f5-c6b2-4ed2-aab5-64c2401ddf65}",
"acrylicOpacity" : 0.75,
"closeOnExit" : true,
"colorScheme" : "Campbell",
"commandline" : "\"%PROGRAMFILES%\\git\\usr\\bin\\bash.exe\" -i -l",
"cursorColor" : "#FFFFFF",
"cursorShape" : "bar",
"fontFace" : "Consolas",
"fontSize" : 10,
"historySize" : 9001,
"icon" : "%PROGRAMFILES%\\Git\\mingw64\\share\\git\\git-for-windows.ico",
"name" : "Bash",
"padding" : "0, 0, 0, 0",
"snapOnInput" : true,
"startingDirectory" : "%USERPROFILE%",
"useAcrylic" : true
},

如果用CMD多点也可以按照以下方法设置CMD为默认终端

设置Git Bash为默认终端

在文件中找到defaultProfile并将defaultProfile花括号中的guid换成自己的

1

"defaultProfile": "{76b973f5-c6b2-4ed2-aab5-64c2401ddf65}",

这时候关掉Windows Terminal再打开默认的已经是Git Bash了.
